# Chromatic Vision

## Product Summary

Chromatic is a portfolio and creative system built around a point-based visual language.

It still has two layers:

- a world layer: train compartment, objects, atmosphere, later character presence
- a site layer: clear, recruiter-friendly, accessible portfolio content

The visual target is persistent colored points on dark grounds, with strong silhouettes, spatial depth, and stable motion.

## Experience Structure

### World Layer

The train-compartment idea remains central.

Important ingredients:

- a compartment or carriage-like space
- a table and meaningful props
- a laptop or portal into the site
- motion outside the window or beyond the scene
- cinematic atmosphere: darkness, glow, travel, weather, vibration

### Site Layer

The site should be easier to read than the world layer, but still borrow the same visual language through:

- hero treatments
- previews
- transitions
- selected project surfaces

Body text, navigation, forms, and other accessibility-critical UI should remain DOM-native.

### Content Mirroring

The mirroring idea survives:

- object or scene detail in the world
- matching page, project, or section in the site

Examples:

- ticket or business card -> contact / links
- paper or dossier -> resume / experience
- tabletop object -> interactive project showcase
- window view -> gallery / browsing
- laptop portal -> functional site

## Artistic Direction

The target look is:

- persistent 3D points or surfels
- colored stipple on black
- subtle variation in size, opacity, and density
- restrained bloom
- legible silhouettes before dense realism
- motion without frame-to-frame swimming

## Source Types

The long-term system should be able to support:

- Blender/glTF models
- animated/skinned characters
- still images
- recorded video
- live webcam/video
- LiDAR or other scan inputs
- later XR presentation layers

Phase 1 does not need all of them.

## Phase 1 Non-Goals

Phase 1 is not:

- a full portfolio site
- a full train experience
- native iOS capture
- XR support
- a universal import stack

Phase 1 is only a proof that the point runtime can carry the look and motion quality the project needs.
